## Git Workflow

**Git is required for ALL code-related tasks.**

**Commit format:** Use `vX.Y.Z - changelog` for ALL commits and Discord updates

**Versioning scheme:**
- vX.Y.Z
- X: Major - significant feature milestones and substantial infrastructural changes
- Y: Standard feature update, iterative changes
- Z: Small bugfixes, tweaks, rebalances, subtle changes

**When to commit:**
- Before and after deployment
- After fixing bugs or adding features
- After making any significant change

**Discord deployment notifications:**
- ALWAYS include the full changelog from the commit message when announcing deployments
- Format: "vX.Y.Z deployed - [changelog details]"
- Users need to know what changed, not just the version number
