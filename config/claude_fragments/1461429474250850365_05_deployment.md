## Deployment - wendy.monster

Deploy static websites or multiplayer game servers to wendy.monster.

```bash
./scripts/wendy/deploy.sh <project-path> [target-url]
```

**Auto-detection based on files:**
- Has `index.html` -> Static site at `wendy.monster/<name>/`
- Has `server.ts` -> Game server at `wendy.monster/game/<name>/` with WebSocket

**Examples:**
```bash
# Deploy landing/ folder to wendy.monster/landing/
./scripts/wendy/deploy.sh landing

# Deploy landing/ to custom URL wendy.monster/my-cool-site/
./scripts/wendy/deploy.sh landing my-cool-site

# Deploy a game server
./scripts/wendy/deploy.sh snake-game
# -> Game at wendy.monster/game/snake-game/
# -> WebSocket at wss://wendy.monster/game/snake-game/ws
```

### Game Server Development

**For any multiplayer game project, ALWAYS read the auto-loaded `multiplayer-game-guide.md`** (injected when game keywords detected). It has full examples, correct API signatures, and debugging steps.

Quick reference: `server.ts` + `public/` folder structure. Helper library: `createGameServer()`, `loadState()`/`saveState()`, `broadcast()`/`send()`. Limits: 256MB memory, 0.5 CPU, max 20 concurrent games.

**Debugging:** `./scripts/wendy/game_logs.sh <game-name> [lines]`
