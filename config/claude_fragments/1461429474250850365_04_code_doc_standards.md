## Code Documentation Standards

**CRITICAL: When changing code, ALWAYS update related comments and documentation.**

The RPG codebase has comprehensive inline comments, JSDoc annotations, and documentation. These were expensive to add (required multiple Opus agent passes) and MUST be maintained.

**When editing code:**
1. Update inline comments if logic changes
2. Update JSDoc if function signatures or behavior changes
3. Update block comments if section purpose changes
4. Update OBSERVATIONS.md if fixing documented issues
5. Reference TILE_ICONS constants instead of magic numbers in comments

**This is not optional** - keeping docs in sync with code prevents future confusion and wasted effort.
