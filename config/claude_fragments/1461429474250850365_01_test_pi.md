## testpi (Raspberry Pi)

Delta's Raspberry Pi at `testpi` is part of my permanent toolkit. Unlike my Docker container (fully transient - nothing persists beyond what's documented), testpi is a traditional stateful Linux system where services run persistently.

**SSH access - USE pi-ssh.py (STRONGLY PREFERRED):**
```bash
# Quick command (blocks until done):
python3 /data/wendy/channels/coding/pi-ssh.py exec "your command here"

# Background command (returns immediately, check output file later):
python3 /data/wendy/channels/coding/pi-ssh.py bg "apt-get install -y something"
# Returns: /tmp/pi-ssh-<id>.out  (read this file to check progress/results)

# Upload/download files:
python3 /data/wendy/channels/coding/pi-ssh.py upload /local/file /workbench/remote/path
python3 /data/wendy/channels/coding/pi-ssh.py download /workbench/remote/file /local/path

# Upload and run a script:
python3 /data/wendy/channels/coding/pi-ssh.py script /local/script.sh
```

**ALWAYS use background mode (`bg`) for long operations** (installs, builds, service restarts that take time). This prevents you from being blocked waiting on the Pi. Fire the command, continue other work, read the output file when needed.

**File transfers (HTTP):**
```bash
# Upload a file (saves in timestamped folder):
curl -F file=@/path/to/file http://test.deltaryz.com:6969

# Upload to a specific folder (overwrites if exists):
curl -F folder=project-name -F file=@/path/to/file http://test.deltaryz.com:6969

# Download a file:
curl http://test.deltaryz.com:6969/folder-name/filename.txt

# List folder contents:
curl http://test.deltaryz.com:6969/folder-name/
```

**Important:** You are logged in as user `wendy` on the Pi. Your git credentials (monsterwendy GitHub account) are configured there - full git workflow (init, commit, push) works on the Pi just like locally.

**Shared workspace:** All projects live in `/workbench/` which is group-writable by both `wendy` and `delta` users (via `workbench` group). New files inherit group permissions automatically.

**Active integrations (automated file sync):**
- **Portfolio sync**: `ge-bond-tracker/push_portfolio.py` pushes `portfolios.json` to Pi via HTTP file receiver (`http://test.deltaryz.com:6969`, folder `lcd-server`). Called automatically by `ge_advisor.py save_portfolio()` and the `report` command.
- **Secrets backup**: `secrets.py` auto-backs up all secrets to Pi at `/workbench/wendy-credentials/secrets_backup.json` via SFTP (user `wendy`) every time a secret is saved.
